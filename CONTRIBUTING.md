# Contributing to Bonzi Buddy Safe

Thank you for your interest in contributing to Bonzi Buddy Safe! We welcome contributions from the community. This document provides guidelines for contributing to the project.

## Code of Conduct
Please be respectful and constructive in all interactions with other contributors.

## How to Contribute

### Reporting Bugs
If you find a bug, please create an issue with:
- A clear description of the bug
- Steps to reproduce it
- Expected vs. actual behavior
- Your environment (OS, Python version, etc.)

### Suggesting Features
We'd love to hear feature suggestions! Please create an issue with:
- A clear description of the feature
- Why you think it would be useful
- Any relevant examples or mockups

### Submitting Code
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Write or update tests as needed
5. Ensure all tests pass (`pytest`)
6. Commit with clear messages (`git commit -m 'Add feature: description'`)
7. Push to your fork (`git push origin feature/your-feature`)
8. Create a Pull Request with a clear description

## Code Standards
- Follow PEP 8 style guidelines
- Write docstrings for all functions and classes
- Include type hints where possible
- Keep functions small and focused
- Write unit tests for new functionality

## Security
If you discover a security vulnerability, please email us privately instead of opening an issue. We take security seriously.
